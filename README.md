# index-home-resource

## 项目简介

这是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或具体网站，仅作为一系列静态 HTML 资源的集中存放与托管空间。每个页面均为独立文件，可单独访问，彼此之间无直接关联。

## 目录说明

```
.
├── index.html          # 仓库入口页（可自定义）
├── pages/              # 存放各独立 HTML 页面的文件夹
│   ├── page-xxx.html
│   └── ...
├── assets/             # 公共资源（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/` 目录下存放所有独立的 HTML 页面文件，文件名建议保持简洁且具有一定可读性。
- `assets/` 目录用于存放多个页面可能共用的静态资源，便于维护和复用。
- 根目录下的 `index.html` 可作为仓库的导航入口，可根据需要自行修改。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，每个页面有自己的完整结构和内容。页面之间不共享数据、不依赖后端服务，仅通过浏览器即可正常渲染。

归档原则：
- 每个页面以独立文件形式存在，便于索引和引用。
- 文件命名遵循小写字母、数字、连字符组合，避免特殊字符。
- 页面内引用的资源路径建议使用相对路径，以确保在 GitHub Pages 或其他静态托管环境中正常工作。

## 维护说明

- 本仓库主要面向个人或小团队维护，用于整理和展示零散的 HTML 页面资源。
- 新增页面时，请将 HTML 文件放入 `pages/` 目录，并视需要调整 `index.html` 的导航链接。
- 如需更新已有页面，请直接修改对应文件，并保持文件命名不变。
- 本仓库不提供自动化构建流程，所有页面均为纯静态文件，直接提交即可生效。
